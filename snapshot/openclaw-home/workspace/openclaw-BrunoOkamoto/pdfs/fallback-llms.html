3⚙️ Configurar fallback no openclaw.json

Essa configuração garante que, se um modelo falhar, o OpenClaw tenta automaticamente o próximo — sem você precisar fazer nada.

Configuração básica com fallback

// ~/.openclaw/openclaw.json
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: [
          "google/gemini-3.1-pro-preview",
          "openai/gpt-5.1-codex"
        ]
      }
    }
  }
}

Com aliases (nomes amigáveis) e allowlist

{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: [
          "google/gemini-3.1-pro-preview",
          "openai/gpt-5.1-codex"
        ]
      },
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
        "anthropic/claude-opus-4-6":   { alias: "Opus" },
        "anthropic/claude-haiku-4-5":   { alias: "Haiku" },
        "google/gemini-3.1-pro-preview": { alias: "Gemini" },
        "openai/gpt-5.1-codex":          { alias: "GPT" }
      }
    }
  }
}

Comandos CLI para gerenciar fallbacks

# Ver modelo atual e fallbacks configurados
openclaw models status

# Listar todos os modelos disponíveis
openclaw models list

# Trocar modelo primário
openclaw models set anthropic/claude-sonnet-4-6

# Adicionar fallback
openclaw models fallbacks add google/gemini-3.1-pro-preview

# Listar fallbacks
openclaw models fallbacks list

# Remover fallback
openclaw models fallbacks remove openai/gpt-5.1-codex

# Limpar todos os fallbacks
openclaw models fallbacks clear

# Adicionar alias (nome amigável)
openclaw models aliases add Gemini google/gemini-3.1-pro-preview

Configurar via config set (sem editar arquivo)

# Definir modelo primário pela linha de comando
openclaw config set agents.defaults.model.primary "anthropic/claude-sonnet-4-6"

# Ver a config ativa
openclaw config get agents.defaults.model

4🔑 Adicionar credenciais de novos provedores

Anthropic (API key)

openclaw onboard --auth-choice anthropic-api-key

Ou cole direto:

openclaw models auth paste-token \
  --provider anthropic

Google Gemini (API key)

openclaw onboard --auth-choice gemini-api-key

Ou via variável de ambiente:

export GEMINI_API_KEY=sua_chave

OpenAI (API key)

openclaw onboard --auth-choice openai-api-key

Gemini CLI (grátis, OAuth)

openclaw plugins enable google-gemini-cli-auth
openclaw models auth login \
  --provider google-gemini-cli \
  --set-default
⚠️ O Gemini CLI OAuth é uma integração não oficial. Use uma conta Google não crítica. O Google pode restringir contas que utilizam clientes de terceiros.

5🩺 openclaw doctor — diagnóstico e correção

Sintomas de que você precisa do doctor: agente não responde, erro de modelo, gateway offline, configuração corrompida após trocar LLM.

Via agente (mais fácil)

💬 Prompt — diagnóstico básico
Roda o openclaw doctor e me fala o que encontrou
💬 Prompt — com correção automática
Roda openclaw doctor --fix e corrige os problemas encontrados
💬 Prompt — varredura profunda
Faz uma varredura completa do sistema com openclaw doctor --deep --fix

Via terminal

🩺 Comandos do Doctor
# Verificação básica de saúde
openclaw doctor

# Verificar E aplicar correções recomendadas
openclaw doctor --fix

# Varredura profunda do sistema
openclaw doctor --deep

# Correção profunda sem confirmações (automático)
openclaw doctor --deep --fix --yes

# Correção forçada (sobrescreve configurações customizadas)
openclaw doctor --fix --force

# Regenerar gateway token se perdido
openclaw doctor --generate-gateway-token

# Modo não interativo (pra scripts/automação)
openclaw doctor --fix --non-interactive

Quando usar cada comando

ProblemaComando
Agente não responde após trocar LLM openclaw doctor --fix
Gateway offline openclaw doctor --deep --fix
Config corrompida / inválida openclaw doctor --fix --yes
Erro "model not allowed" após mudança openclaw doctor --fix + verificar allowlist
Token do gateway sumiu openclaw doctor --generate-gateway-token
Tudo quebrado, quero reset total openclaw doctor --deep --fix --force --yes

Validar a config antes de aplicar

# Valida o openclaw.json sem reiniciar o gateway
openclaw config validate

# Ver o caminho do arquivo de config ativo
openclaw config file

6🚨 Fluxo de emergência: tokens acabaram

Se os créditos do Anthropic acabarem e você não tiver outro modelo configurado, o agente para de responder. Siga esse fluxo para resolver via terminal. 
# Passo 1: Verificar o status atual
openclaw models status

# Passo 2: Adicionar Gemini como fallback imediato
openclaw models fallbacks add google/gemini-3.1-pro-preview

# Passo 3: Trocar o primário para Gemini agora
openclaw models set google/gemini-3.1-pro-preview

# Passo 4: Reiniciar o gateway para aplicar
openclaw gateway restart

# Passo 5: Verificar se está funcionando
openclaw doctor

Opção grátis: Gemini CLI (sem API key)

# Ativar plugin do Gemini CLI
openclaw plugins enable google-gemini-cli-auth

# Fazer login com conta Google (abre navegador)
openclaw models auth login --provider google-gemini-cli --set-default

# Trocar pra Gemini CLI
openclaw models set google-gemini-cli/gemini-2.5-pro

# Reiniciar
openclaw gateway restart
💡 Para evitar essa situação: configure fallbacks antes de precisar. O OpenClaw trocará automaticamente quando detectar que os créditos acabaram.
