Seção 1
Por que usar o 1Password?

Antes de começar a configurar, é importante entender por que usar um gerenciador de segredos em vez de guardar as chaves em arquivos .env ou no código. A resposta curta: segurança, conveniência e profissionalismo.

Aspecto 🗂️ Arquivo .env 🔐 1Password CLI
Segurança ❌ Risco alto
Fácil de vazar via git, logs ou backups		 ✅ Cofre criptografado
AES-256, zero-knowledge
Compartilhamento ❌ Manual
Copiar/colar, WhatsApp, Slack...		 ✅ Vault compartilhado
Acesso controlado por equipe
Rotação de chaves ❌ Trabalhoso
Editar arquivos em cada máquina		 ✅ Atualiza em um lugar
Todos os projetos usam automaticamente
Auditoria ❌ Sem histórico
Quem acessou? Impossível saber		 ✅ Logs de acesso
Histórico completo de quem usou
Integração com CI/CD ❌ Variáveis manuais ✅ Integração nativa
GitHub Actions, GitLab CI, etc
⚠️
Atenção

Arquivos .env com API keys já foram responsáveis por prejuízos de milhares de dólares para desenvolvedores desatentos — uma chave da OpenAI commitada por engano pode gerar uma fatura enorme antes que você perceba.

Seção 2
Criar sua conta no 1Password
Passo 1 de 2 · Conta
1

Acesse o site oficial

Abra o navegador e vá para 1password.com/pt. Clique em "Experimente grátis" — o plano Individual é o suficiente para o curso e custa ~US$2.99/mês após o trial de 14 dias.

2

Complete o cadastro e guarde a Secret Key

Após criar a conta, o 1Password vai gerar sua Emergency Kit (Kit de Emergência) — um PDF com seu Secret Key. Imprima ou salve esse PDF em local seguro. Sem ele, você não consegue recuperar acesso.

❌
Erro comum

Nunca salve o Emergency Kit em serviços de nuvem não criptografados (Google Drive, Dropbox em texto plano). Imprima fisicamente ou use outro cofre criptografado.

💡
Dica

O plano Teams ou Families permite compartilhar vaults com colegas de projeto. Se você trabalha em equipe, considere esses planos desde o início.

Seção 3 e 4
Instalar o App e o CLI
Passo 2 de 6 · Instalação

3.1 — Instalar o App Desktop

O app desktop é necessário para a integração com o CLI funcionar (ele atua como o cofre local que o CLI consulta sem precisar de senha toda vez).

🍎
macOS
Baixe em 1password.com/downloads ou pela Mac App Store
🪟
Windows
Baixe o instalador .exe em 1password.com/downloads
🐧
Linux
Pacote .deb/.rpm disponível em 1password.com/downloads/linux
💡
Dica — Linux

No Linux, instale via repositório oficial para receber atualizações automáticas. O 1Password tem suporte nativo para Ubuntu, Fedora e Arch.

3.2 — Instalar o CLI op

O CLI op é o programa que você usará no terminal e que o OpenClaw chamará automaticamente para buscar suas credenciais.

bash · macOS
# macOS via Homebrew (recomendado)
brew install 1password-cli

# Verificar instalação
op --version
bash · Linux
# Adicionar chave GPG do repositório
curl -sS https://downloads.1password.com/linux/keys/1password.asc | \
  sudo gpg --dearmor --output /usr/share/keyrings/1password-archive-keyring.gpg

# Adicionar repositório apt
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/1password-archive-keyring.gpg] https://downloads.1password.com/linux/debian/$(dpkg --print-architecture) stable main" | \
  sudo tee /etc/apt/sources.list.d/1password.list

# Instalar
sudo apt update && sudo apt install 1password-cli
powershell · Windows
# Windows via winget
winget install AgileBits.1Password.CLI

# Verificar (abra um novo terminal após instalar)
op --version
⚠️
Atenção — macOS com Homebrew

No macOS, o Homebrew instala o op em /opt/homebrew/bin/op (Apple Silicon) ou /usr/local/bin/op (Intel). Anote esse caminho — você vai precisar dele na configuração do OpenClaw.

Integração Desktop e Primeiros Comandos
Passo 3 de 6 · Configuração

5 — Habilitar Integração Desktop

Esta etapa é o segredo para que o CLI não peça senha toda vez — ele usa o app desktop como autenticador. Sem isso, você precisará fazer login via terminal toda vez.

1Password · Preferências
⚙️ Navegue até Settings → Developer → 🔌 Integrate with 1Password CLI
✓ Integrate with 1Password CLI — habilite este toggle
Connect with SSH Agent (opcional)
🍎 macOS: Settings → Developer  |  🪟 Windows: habilite Windows Hello primeiro  |  🐧 Linux: Settings → Security primeiro
⚠️
Windows — Pré-requisito

No Windows, é necessário habilitar Windows Hello (reconhecimento facial, digital ou PIN) antes de ativar a integração com o CLI.

⚠️
Linux — Pré-requisito

No Linux, vá primeiro em Settings → Security e habilite "Unlock using system authentication". Depois vá em Settings → Developer para integrar o CLI.

6 — Primeiros Comandos no Terminal

Com o app instalado e a integração habilitada, abra o terminal e teste:

bash
# Ver sua conta ativa
op whoami

# Listar seus vaults (cofres)
op vault list

# Listar itens em um vault
op item list --vault "Private"

# Se não estiver logado, faça login explícito
op signin
Terminal · bash
$ op whoami
Email:   seu@email.com
Account: minha-conta
UserUUID: ABCDEF123...

$ op vault list
ID                      NAME
abc123def456...   Private
xyz789ghi012...   Shared
✓ CLI funcionando corretamente!
✅
Sucesso

Se op whoami mostrou seu e-mail e op vault list listou seus vaults — parabéns! A integração está funcionando. O 1Password app autentica em background usando biometria ou PIN.

Seção 7 e 8
Criar e Ler Credenciais
Passo 4 de 6 · Credenciais

7 — Criar um Item de API Key

Agora vamos guardar sua chave da Anthropic (ou OpenAI) no 1Password. Você pode fazer isso pela interface gráfica ou pelo CLI.

Opção A — Interface gráfica (mais fácil para iniciantes):

1Password · Vault Private
1️⃣ Clique em + New Item no canto superior direito
2️⃣ Selecione 🔌 API Credential na lista de tipos
3️⃣ Título: Anthropic API Key  |  Campo credential: cole sua chave sk-ant-api03-...
4️⃣ Clique em 💾 Save

Opção B — Via CLI (mais rápido se você já está no terminal):

bash
# Criar item de API credential no vault Private
op item create \
  --category=api-credential \
  --title="Anthropic API Key" \
  --vault="Private" \
  credential=sk-ant-api03-XXXXX

# Para OpenAI:
op item create \
  --category=api-credential \
  --title="OpenAI API Key" \
  --vault="Private" \
  credential=sk-proj-XXXXX

8 — Ler Credenciais via CLI

Existem duas formas de ler suas credenciais — com op item get para inspeção ou com op read para uso em scripts.

bash
# Ler campo específico (com reveal para ver o valor)
op item get "Anthropic API Key" --field credential --reveal

# Usar via URI op:// (formato Secret Reference)
op read "op://Private/Anthropic API Key/credential"

# Usar diretamente em um export de variável de ambiente
export ANTHROPIC_API_KEY=$(op read "op://Private/Anthropic API Key/credential")

# Confirmar que funcionou
echo $ANTHROPIC_API_KEY
💡
Secret Reference URI

O formato op://Vault/Item/Field é chamado de Secret Reference. É a forma mais segura de referenciar credenciais — o valor nunca fica exposto no código, apenas o caminho para ele.

❌
Erro comum — Vault com espaços

Se seu vault se chama My Vault (com espaço), o URI fica op://My Vault/Item/Field. Se der erro, tente usar o ID numérico do vault: op vault list mostra os IDs.

Seção 9
Integrar com o OpenClaw
Passo 5 de 6 · Integração

Com o 1Password funcionando, é hora de conectar tudo ao OpenClaw. O OpenClaw suporta um sistema chamado SecretRef exec — em vez de guardar a chave no openclaw.json, você configura o OpenClaw para chamar o op CLI em tempo real para buscar a credencial.

9.1 — Configuração no openclaw.json

json · openclaw.json
{
  "secrets": {
    "providers": {
      "onepassword": {
        "source": "exec",
        "command": "/usr/local/bin/op",
        "args": ["read", "op://Private/Anthropic API Key/credential"],
        "passEnv": ["HOME"],
        "jsonOnly": false
      }
    }
  },
  "models": {
    "providers": {
      "anthropic": {
        "apiKey": {
          "source": "exec",
          "provider": "onepassword",
          "id": "value"
        }
      }
    }
  }
}

9.2 — macOS com Homebrew (caminho diferente)

No macOS com Homebrew (especialmente Apple Silicon), o op fica em um caminho diferente e é um symlink. Adicione allowSymlinkCommand:

json · macOS Homebrew
{
  "source": "exec",
  "command": "/opt/homebrew/bin/op", // Apple Silicon
  "allowSymlinkCommand": true,
  "trustedDirs": ["/opt/homebrew"],
  "args": ["read", "op://Private/Anthropic API Key/credential"],
  "passEnv": ["HOME"],
  "jsonOnly": false
}
⚠️
Descobrir o caminho do op

Para descobrir onde o op está instalado: which op no macOS/Linux. No Windows: where op no CMD ou Get-Command op no PowerShell.

9.3 — Alternativa Simples (sem SecretRef)

Para iniciantes que querem usar o 1Password só como cofre manual e configurar o ambiente antes de iniciar o OpenClaw:

bash
# Exportar a variável usando o op CLI
export ANTHROPIC_API_KEY=$(op read "op://Private/Anthropic API Key/credential")

# Reiniciar o gateway com a chave disponível
openclaw gateway restart
💡
Dica — Adicione ao seu shell profile

Adicione o export acima ao seu ~/.zshrc ou ~/.bashrc para que a variável seja sempre carregada ao abrir o terminal. A chave nunca fica salva no arquivo — o op busca do cofre em tempo real.

Seção 10
Fluxo Completo: openclaw secrets
Passo 6 de 6 · Ativação

O OpenClaw tem um comando dedicado para gerenciar segredos. O fluxo completo é simples — são 5 comandos em sequência:

🔍
Auditar
secrets audit
→
🛠️
Configurar
secrets configure
→
✅
Aplicar
secrets apply
→
🔄
Recarregar
secrets reload
→
🎯
Verificar
audit --check
bash · Sequência completa
# 1. Auditar o estado atual das credenciais
# Mostra quais secrets estão configurados e quais faltam
openclaw secrets audit --check

# 2. Iniciar configuração interativa
# Guia você passo a passo para cada secret necessário
openclaw secrets configure

# 3. Aplicar a configuração gerada
openclaw secrets apply --from /tmp/openclaw-secrets-plan.json

# 4. Recarregar o runtime para usar os novos secrets
openclaw secrets reload

# 5. Verificar novamente — deve mostrar tudo verde
openclaw secrets audit --check
Terminal · openclaw secrets audit --check
$ openclaw secrets audit --check

Checking secret providers...

✓ anthropic.apiKey     → resolved via exec (onepassword)
✓ openai.apiKey       → resolved via exec (onepassword)
· slack.botToken      → not configured (optional)

✅ All required secrets resolved!
✅
Sucesso

Quando o audit mostrar todos os secrets obrigatórios como ✓ resolved, o OpenClaw está pronto para usar as credenciais do 1Password automaticamente — sem você precisar fazer nada toda vez que iniciar.

❌
Erro — "op: command not found" ou "exec failed"

Isso significa que o caminho do op no openclaw.json está errado. Confirme com which op e atualize o campo "command" com o caminho completo correto.

Checklist Final

Use esta lista para confirmar que tudo está configurado corretamente antes de começar a usar o OpenClaw com 1Password:

  • □
    Conta criada no 1Password
    Você se registrou em 1password.com e salvou seu Emergency Kit em local seguro
  • □
    App desktop instalado e logado
    O app está instalado no seu OS, você está logado e consegue ver seus vaults
  • □
    CLI instalado e verificado
    op --version retorna uma versão válida (ex: 2.x.x)
  • □
    Integração desktop habilitada
    Settings → Developer → "Integrate with 1Password CLI" está ativado
  • □
    op whoami funciona sem pedir senha
    O CLI autentica via biometria/PIN do app, sem digitar master password
  • □
    API Keys guardadas no 1Password
    Anthropic, OpenAI ou outras chaves estão como itens "API Credential" no vault Private
  • □
    op read funciona
    op read "op://Private/Anthropic API Key/credential" retorna a chave sem erro
  • □
    openclaw.json configurado com SecretRef exec
    O campo command aponta para o caminho correto do op (use which op para confirmar)
  • □
    openclaw secrets audit --check passa verde
    Todos os secrets obrigatórios aparecem como "resolved"
  • □
    OpenClaw gateway funcionando
    openclaw gateway status mostra running e o agente responde normalmente
✅
Parabéns!

Se todos os itens estão marcados, você configurou o 1Password com o OpenClaw com sucesso. Suas API keys estão protegidas em um cofre criptografado, acessadas automaticamente quando necessário — zero .env, zero risco de vazar chaves.
