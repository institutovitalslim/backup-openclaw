🎁 Boa notícia: seu agente já sabe navegar na internet.

Assim que você instala o OpenClaw, duas ferramentas de busca já estão disponíveis — sem configuração, sem API key, sem custo extra. São as primeiras coisas que o seu agente vai usar quando você pedir pra ele pesquisar alguma coisa.

🔍
GRÁTIS · NATIVO
web_search
Busca na web via Brave integrado. Retorna títulos, URLs e snippets das páginas mais relevantes para uma query.
✅ Busca em tempo real
✅ Filtro por data, idioma e região
✅ Sem API key necessária
📄
GRÁTIS · NATIVO
web_fetch
Lê o conteúdo completo de qualquer URL — artigo, página de produto, landing page, documentação. Extrai o texto limpo.
✅ Qualquer URL acessível
✅ HTML → texto legível
✅ Funciona com markdown

Subindo o nível: Brave Search API própria

🦁 Por que usar sua própria API key do Brave?

A busca nativa do OpenClaw é compartilhada — funciona, mas tem limites. Com sua própria key, você tem mais resultados por busca, sem filtros artificiais, e controle total sobre volume.

⚙️ Como configurar — 3 passos simples:

1
Crie sua conta e pegue a API key
Acesse brave.com/search/api → crie uma conta → vá em "API Keys" → clique em "New Key". O plano gratuito dá 2.000 buscas/mês — suficiente pra maioria dos founders.
2
Configure no OpenClaw
Com a key em mãos, rode o comando abaixo no seu terminal:
# Configure sua Brave API Key no OpenClaw
openclaw config set brave.apiKey BSA-xxxxxxxxxxxxxxxxxxxxxxxx

# Para confirmar que foi salva:
openclaw config get brave.apiKey
3
Pronto — seu agente já usa a nova key
A partir de agora, toda vez que o agente usar web_search, ele vai bater na sua própria cota. Você consegue monitorar o consumo direto no painel da Brave.
📊
2.000 buscas/mês — o que isso significa na prática?

Se o seu agente faz 3-5 pesquisas por dia (monitoramento de concorrentes, notícias, etc.), você fica bem dentro do limite gratuito. Para uso mais intenso, o plano pago começa em $3/1.000 buscas — ainda barato.

3
Pesquisa profunda: Perplexity API

🧠 Qual a diferença entre web_search e Perplexity?

O web_search te dá links e snippets. O Perplexity te dá análise. É um modelo de IA que já leu as páginas, cruzou as informações e te entrega uma resposta sintetizada com fontes. Quando você quer entender algo, não só achar links — é Perplexity.

🔍
web_search — use quando:
  • Quer achar uma URL específica
  • Monitorar se saiu novidade
  • Busca rápida de referência
  • Checar preço / contato de empresa
🧠
Perplexity — use quando:
  • Quer entender um mercado
  • Pesquisa de contexto profundo
  • Análise de múltiplas fontes
  • Quer conclusão, não lista de links

⚙️ Como configurar o Perplexity:

1
Crie a conta e gere a API key
Acesse perplexity.ai → Settings → API → "Generate". O Perplexity usa créditos pré-pagos — você carrega $5, $10 ou mais e gasta conforme usa.
2
Salve no 1Password
Não salve a key direto no config. Guarde no 1Password com o nome "Perplexity API" — seu agente busca de lá quando precisar.
# Salvar no 1Password (via CLI do op)
op item create \
  --category "API Credential" \
  --title "Perplexity API" \
  --vault "Amora Vault" \
  "credential=pplx-xxxxxxxxxxxxxxxxxxxxxxxx"

# Para buscar depois (o agente faz isso automaticamente):
op item get "Perplexity API" --field credential --reveal
3
Pronto — chame via skill
A skill de Perplexity no seu workspace já sabe buscar a key no 1Password. Basta dizer "usa o Perplexity pra pesquisar X" ou configurar o deep research (veja o bônus abaixo).

# Deep Research Protocol

## Protocolo (executar em ordem)

### 1. ENQUADRAMENTO (2 min)
- Qual a pergunta exata que estou respondendo?
- Qual decisão essa pesquisa vai informar?
- Profundidade: quick scan (5-10 min) | standard (15-30 min) | deep dive (30-60 min)?

### 2. COLETA MULTI-FONTE
Fontes primárias (nesta ordem):
1. web_search    — varredura geral (Brave)
2. web_fetch     — extrair conteúdo das URLs mais relevantes
3. Perplexity API — síntese com AI e citações (key no 1Password: "Perplexity API")
4. Knowledge Base — se tiver conteúdo relevante já ingestado

Fontes especializadas (quando relevante):
- Reddit/HackerNews — opinião de comunidade técnica
- Twitter/X — sinais em tempo real
- YouTube — análises longas

### 3. SÍNTESE
Para cada fonte, classificar:
- FATO    → confirmável, com fonte
- OPINIÃO → interpretação, marcar como tal
- SINAL   → sugere tendência, monitorar

### 4. RELATÓRIO (formato padrão)

# [Tema] — Deep Research
Data: DD/MM/YYYY

## TL;DR
- [3-5 bullets com o essencial]

## Contexto
[Por que estamos pesquisando / qual decisão isso informa]

## Findings
### [Subtema 1]
- Finding + fonte
### [Subtema 2]
- Finding + fonte

## Análise
[Cruzamento dos findings, padrões, o que significa]

## Recomendação
[O que fazer com essa informação]

## Fontes
1. [URL 1]
2. [URL 2]

### 5. SALVAMENTO
- Salvar em memory/research-YYYY-MM-DD-[slug].md

## Níveis de profundidade
| Quick scan  | 5-10 min  | 3-5 URLs   | 1 parágrafo + bullets  |
| Standard    | 15-30 min | 8-12 URLs  | Relatório 1-2 páginas  |
| Deep dive   | 30-60 min | 15-20 URLs | Relatório 3-5 páginas  |

## Vieses a evitar
- Confirmation bias → buscar ativamente contra-argumentos
- Recency bias → verificar se tendências são reais ou hype
- Survivorship bias → buscar casos de fracasso, não só sucesso
