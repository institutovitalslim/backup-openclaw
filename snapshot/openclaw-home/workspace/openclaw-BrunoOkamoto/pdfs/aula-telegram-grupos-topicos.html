01
O que são tópicos?
Entendendo a estrutura antes de configurar

Imagine um grupo do Telegram como um prédio. Tópicos são os andares — cada andar tem sua própria conversa, mas todo mundo mora no mesmo prédio.

Com tópicos, você pode ter no mesmo grupo:

  • Um tópico para o curso — onde seu agente tira dúvidas dos alunos
  • Um tópico de suporte técnico — sem misturar com o restante
  • Um tópico para ideias e projetos
  • Um tópico de uso pessoal, só pra você
💡
A grande vantagem é que você pode ter o mesmo agente se comportando diferente em cada tópico — ou até usar agentes completamente separados por tópico.
02
Criando seu grupo com tópicos
Passo a passo do zero
⚠️
Atenção: Só supergrupos suportam tópicos. Um grupo normal não tem essa opção. O passo 3 abaixo transforma seu grupo em supergrupo automaticamente.
1
Crie um novo grupo no Telegram
Menu (☰) → Novo Grupo → escolha o nome (ex: Meu HQ) → adicione pelo menos uma pessoa → confirme.
2
Abra as configurações do grupo
Toque no nome do grupo no topo da tela → ícone de lápis (editar).
3
Torne-o um Supergrupo
Vá em Tipo do Grupo → defina um @username público (ex: @meuhq_2025). O Telegram converte automaticamente para supergrupo.
4
Ative os tópicos
Nas configurações → Tópicos → ligue o toggle. O Telegram cria o tópico Geral automaticamente (ID: 1).
5
Crie seus tópicos
Na tela do grupo → ícone de tópicos → "Criar Tópico" → nome + emoji. Cada tópico terá um ID único (ex: 2638).

03
Adicionando seu agente ao grupo
Duas configurações essenciais antes de tudo

Passo 1 — Desligar o Privacy Mode no BotFather

Por padrão, bots só enxergam mensagens onde são marcados diretamente (@nomedo​bot mensagem). Para o agente ler tudo sem precisar ser marcado, você precisa desligar o Privacy Mode:

1
Abra o @BotFather no Telegram
Digite /mybots → escolha seu bot → Bot Settings → Group Privacy → Turn off.
2
Confirme que está desligado
O BotFather vai mostrar: "Privacy mode is disabled for [seu bot]". Pronto.

Passo 2 — Adicionar como Administrador

1
Adicione o bot ao grupo
Configurações do grupo → Administradores → Adicionar Administrador → busque pelo @username do seu bot → adicione.
2
Ative a permissão de tópicos
Na tela de permissões do bot, ative "Gerenciar Tópicos". Sem isso, ele não consegue agir dentro dos tópicos.
🚨
Erro clássico: Adicionar o bot ao grupo mas esquecer de torná-lo admin. Resultado: ele não lê as mensagens dos tópicos, só do "Geral". Sempre torne-o administrador.
04
Descobrindo o ID de cada tópico
A forma mais fácil: peça pra sua própria IA!

Cada tópico tem um ID único que você vai precisar para a configuração. Não precisa de terminal — basta pedir ao seu agente:

No chat privado com seu agente, mande:

"Amora, entra no grupo X, manda uma mensagem qualquer no tópico Y e me diz qual é o chat ID completo desse tópico."

Ela vai responder algo como: telegram:-1003873964847:topic:2638

Anote o ID de cada tópico que você quer configurar. Você vai precisar deles no próximo passo.

05
Configurando como o agente responde
Definindo em quais tópicos ele fala livremente e em quais só quando chamado

Existem dois modos de ativação:

  • mention — o agente só responde quando alguém digita @nomedo​bot mensagem
  • all — o agente responde a qualquer mensagem no tópico, mesmo sem ser marcado

Você define isso por tópico no seu config.yaml:

yamlagents: - id: amora-main model: anthropic/claude-sonnet-4-6 workspaceDir: /root/.openclaw/workspace-amora activation: surfaces: - surface: telegram mode: mention # padrão: só quando marcada overrides: # Tópico "Curso" — responde tudo, sem precisar marcar - chat: "telegram:-1003873964847:topic:2638" mode: all # Tópico "Suporte" — também responde tudo - chat: "telegram:-1003873964847:topic:2640" mode: all # Tópico "Geral" — só quando marcada - chat: "telegram:-1003873964847:topic:1" mode: mention
⚠️
Cuidado com o Chat ID: use sempre o ID com :topic:XXXX no final. Se usar só o ID do grupo (telegram:-1003873964847 sem o tópico), o agente vai responder em todos os tópicos do grupo.

Depois de editar o config, reinicie o gateway para as mudanças entrarem em vigor:

bashopenclaw gateway restart
✅
Testando: mande uma mensagem num tópico configurado com mode: all sem marcar o bot. Se ele responder, funcionou!

06
Um agente ou vários?
A decisão mais importante ao trabalhar com múltiplos tópicos

Quando você tem vários tópicos, precisa decidir: um único agente responde em todos, ou cada tópico tem o seu próprio agente?

🔵 Um agente para tudo
(MAIN compartilhado)
📁 1 workspace compartilhado
🧠 Lembra de tudo de todos os tópicos
⏰ 1 conjunto de crons e heartbeats
💰 Mais barato (1 processo)
⚙️ Setup simples
🟢 Um agente por tópico
(Agentes isolados)
📁 Workspace separado por agente
🔒 Contexto isolado por tópico
⏰ Crons independentes
💰 Mais caro (N processos)
⚙️ Setup mais complexo
✅ Um agente — Vantagens
Memória cruzada ("lembra daquela ideia do outro tópico?")
Setup rápido — 1 config, 1 workspace
1 heartbeat faz tudo
Econômico
✗ Um agente — Riscos
Contexto de tópicos diferentes se mistura
Sem privacidade entre tópicos
Contexto cresce muito rápido
✅ Agentes isolados — Vantagens
Privacidade total (tópico A não vaza no B)
SOUL.md diferente por tópico
Modelo diferente por tópico (Haiku no suporte, Opus no dev)
Contexto limpo e focado
✗ Agentes isolados — Riscos
Agentes não se "lembram" de outros tópicos
Múltiplos crons e heartbeats para manter
Mais configuração inicial
🎯
Recomendação para quem está começando: comece com um único agente. É mais simples, funciona muito bem para uso pessoal, e você pode migrar para agentes isolados quando sentir a necessidade real.

07
Qual arquitetura é pra mim?
Cenários reais para te ajudar a decidir
🔵 Use um agente para tudo quando...
Você é o único usuário dos tópicos. Os tópicos são projetos seus relacionados (blog, curso, app). Você quer que o agente lembre de conversas entre tópicos. Quer manter as coisas simples no começo.
🟢 Use agentes isolados quando...
Você tem múltiplos clientes e precisa que os dados de um nunca apareçam para o outro. Quer comportamentos muito diferentes por tópico (professora em um, suporte técnico em outro). Precisa de modelos diferentes por contexto para controlar custo.
⚡ Use híbrido quando...
Quer um agente principal para uso pessoal (chat privado com você) e agentes isolados para outros contextos — como um tópico compartilhado com clientes ou uma comunidade.
