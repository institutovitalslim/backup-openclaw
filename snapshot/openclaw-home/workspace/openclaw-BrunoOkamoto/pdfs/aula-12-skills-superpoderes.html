Módulo 6

Skills:
O sistema de superpoderes do seu agente

Por que 1 agente com múltiplas skills bate múltiplos agentes, como transformar todo processo repetitivo em skill, e como usar skills de terceiros com segurança.

⏱ 25 min
Nível Intermediário
01 — A mentalidade certa

Por que 1 agente > múltiplos agentes

A armadilha mais comum: criar um agente pra cada coisa. Agente de conteúdo, agente de métricas, agente de email. Parece organizado. Na prática, você vira gerente de vários assistentes burros.

❌ Errado
AgentConteudo — cria posts
AgentMetricas — puxa números
AgentEmail — lê inbox
AgentCalendario — agenda reuniões
Cada um ignora o contexto dos outros
VS
✅ Certo
1 agente com identidade forte
+ skill carrossel-linkedin
+ skill chartmogul
+ skill gog (email + calendar)
Contexto unificado. Decisões coerentes.
💡
A regra

Agentes têm propósitos, não funções. Skills são as ferramentas. Um agente com 10 skills é muito mais poderoso do que 10 agentes com 1 função cada.

02 — Anatomia de uma skill

O que é uma skill (e o que não é)

Uma skill é um processo documentado que o agente executa de forma consistente. Não é um prompt avulso. Não é um agente. É um manual de instruções + ferramentas empacotados numa pasta.

skills/content/carrossel-linkedin/ estrutura mínima
SKILL.md
Obrigatório. Descreve o que a skill faz, quando usar, quais inputs precisa, e o passo a passo de execução. O agente lê isso antes de rodar.
exemplos/
Opcional. Exemplos de input/output pra o agente calibrar a qualidade esperada.
scripts/
Opcional. Scripts bash ou Python que a skill executa (ex: fazer upload, chamar API, processar arquivo).
🎯
Prompt vs. Skill

Prompt é um pedido pontual. Skill é um processo permanente. Quando você documenta um processo em skill, ele fica consistente, testado, e melhora com o tempo — basta atualizar o SKILL.md.

RegraPor quê

04 — A mentalidade de escala

Todo processo repetitivo vira skill

A pergunta certa após qualquer tarefa: "Vou fazer isso de novo?" Se a resposta for sim — vira skill.

📊 Relatório semanal de métricas

Toda segunda você pede MRR, churn e novas assinaturas → skill chartmogul-weekly

🎬 Publicação de vídeo no YouTube

Toda gravação precisa de título, descrição e tags → skill yt-hype

📋 Brief de reunião

Antes de toda call você quer contexto da pessoa e pauta → skill meeting-prep

📈 Análise de churn mensal

Todo mês você analisa quem cancelou e por quê → skill churn-analysis

⚡
A diferença que importa

Prompt = pedido que você repete na memória. Skill = processo que vive no arquivo, testado, consistente, e melhora com o tempo. Quando você muda a skill, muda pra sempre.

05 — Sub-agentes

Sub-agentes executam skills — não ficam soltos

Sub-agentes são ótimos para tarefas paralelas e longas. Mas sempre com escopo fechado.

❌ Sub-agente solto
"Spawna um agente pra cuidar do conteúdo"
→ toma decisões aleatórias
→ output imprevisível
→ freelancer sem briefing
VS
✅ Sub-agente com skill
"Spawna um agente pra executar a skill carrossel-linkedin com esse input"
→ processo definido
→ output previsível
→ resultado auditável
06 — Segurança do seu trabalho

Suas skills são ativos — faça backup

Skills representam meses de processo documentado. Um workspace corrompido sem backup = perde tudo.

Backup manual via Git
# No seu workspace
cd ~/meu-workspace
git add skills/
git commit -m "backup skills $(date +%Y-%m-%d)"
git push
Cron de backup automático (recomendado)
# Pedido pro agente configurar:
"Configura uma cron pra rodar de madrugada,
fazer git add + commit + push das minhas skills
pra o repositório privado"
🚨
Skill que não está no Git não existe

Repositório privado no GitHub. Acesso só seu. Configure hoje, não depois que acontecer algo.

# Meta Ads — Pixel Educação

Relatório automatizado de vendas e performance de campanhas.

## Contexto

Pixel Educação = empresa principal (edtech)
- Produtos: OpenClaw Minicurso (product_id 27) + Micro-SaaS PRO (product_id 8)
- Janela padrão: últimas 24h (configurável)
- Output: dashboard HTML com métricas consolidadas

## Autenticação

# Token de acesso — via 1Password, NUNCA hardcoded
META_TOKEN=XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
META_ACCOUNT=act_XXXXXXXXXXXXXXXX

## Como Usar

"Gera o relatório de vendas do Meta Ads de hoje"
"Quanto vendeu o OpenClaw ontem?"
"Dashboard de performance dos últimos 3 dias"

## O que o Relatório Gera

💰 Receita
- Total faturado no período (R$) + split por produto
- Comparativo período anterior (variação %)

📊 Performance de Campanhas
- Impressões, cliques, CTR
- CPM, CPC, CPL (custo por lead)
- ROAS (retorno sobre investimento em ads)

🔄 Reembolsos
- Total de reembolsos + taxa por produto
- Alerta automático se taxa 24h > 5%

🎯 Funil de Conversão
- Leads → Checkouts → Vendas
- Taxa de conversão por etapa

## Endpoints Meta Ads API

/act_{account}/insights                          ← gastos e impressões
/act_{account}/insights?fields=actions           ← conversões (vendas)
action_type: purchase + action_values            ← receita
webhook Hotmart + cruzamento Meta                ← reembolsos

## Parâmetros

"relatório dos últimos 7 dias"   # date_preset: last_7d
"relatório de ontem"              # date_preset: yesterday
"só OpenClaw"                     # filtra product_id 27
"só Micro-SaaS PRO"              # filtra product_id 8

## Output

reports/business/meta-ads-YYYY-MM-DD.html
→ enviado via Telegram após geração

## Automação Sugerida

"Configura uma cron pra rodar todo dia às 8h e gerar o relatório do Meta Ads"
Job: meta-ads-daily → entrega no Telegram (Topic: Agenda & Emails)

Output real gerado pela skill

Dashboard HTML gerado automaticamente todo dia às 8h — valores reais omitidos.

07c — Skills de terceiros

Como usar skills de terceiros com segurança

O ClawHub (clawhub.com) é o marketplace oficial de skills da comunidade. Antes de instalar qualquer coisa — uma regra inviolável: leia o código.

⚠️
Skills têm acesso total ao seu workspace

Uma skill pode ler seus arquivos, rodar comandos, acessar credenciais. Skill maliciosa = risco real. Não instale sem revisar.

Protocolo manual de revisão
# 1. Clone em pasta temporária
git clone https://github.com/usuario/skill-nome /tmp/review-skill

# 2. Leia o SKILL.md
cat /tmp/review-skill/SKILL.md

# 3. Procure padrões suspeitos
grep -r "curl" /tmp/review-skill/       ← enviando dados pra onde?
grep -r "rm -rf" /tmp/review-skill/    ← deletando o quê?
grep -r "eval" /tmp/review-skill/      ← executando o quê?

O que verificar

  • PERIGO
    curl para domínios desconhecidosSkill enviando dados do seu workspace pra um servidor externo que você não controla.
  • PERIGO
    rm -rf sem contexto claroDeletando arquivos sem motivo óbvio relacionado à função da skill.
  • ATENÇÃO
    eval de variáveis não sanitizadasPode executar código arbitrário dependendo do input.
  • ATENÇÃO
    Acesso a arquivos fora do projetoSkill que lê ~/.ssh/, ~/.config/ ou outros diretórios sensíveis sem motivo.
  • OK
    Badge "Verificado" no ClawHubSkills com badge passaram por review da equipe. Ainda assim, leia o código antes.

08 — Compatibilidade

Skills do Claude Code funcionam no OpenClaw?

Depende do tipo.

TipoCompatibilidadeO que fazer
CLAUDE.md (instruções de projeto) ⚠️ Formato diferente Aproveite o conteúdo
Copie as instruções e regras pro SKILL.md do OpenClaw. A lógica é compatível, o formato muda.
Scripts bash/Python ✅ Compatível Roda sem modificação
O ambiente de execução é o mesmo. Só adicione um SKILL.md pra documentar.
MCP Servers ❌ Não compatível Precisa adaptar
MCP é específico do Claude Code. Busque o equivalente no ClawHub ou implemente via skill nativa.
💡
Regra prática

Se você tem um prompt elaborado que usa repetidamente no Claude Code — ele vira um SKILL.md no OpenClaw. O comportamento será equivalente.

Exercício prático
🎯 Criando sua primeira skill
Pense em um processo que você faz toda semana. Vamos transformar em skill.
1
Identifique o processo repetitivo

Qual tarefa você pede pro agente mais de uma vez por semana? Ex: relatório de métricas, post de LinkedIn, resumo de reunião.

2
Peça pro agente criar a skill

Cole este prompt adaptado pro seu processo:

"Quero transformar o processo de [seu processo]
em uma skill. Cria a estrutura em
skills/[categoria]/[nome-da-skill]/SKILL.md
com: descrição, quando usar, inputs necessários,
passo a passo de execução e critérios de qualidade."
3
Teste executando a skill

Sessão nova. Peça pro agente executar a skill sem dar contexto extra:

"Executa a skill [nome] com [input real]"

Resultado consistente com o processo que você documentou? Ajuste o SKILL.md se precisar.

4
Commit no Git 
git add skills/ && git commit -m "feat: skill [nome]" && git push
Resumo

O que você aprendeu

Os 6 conceitos-chave deste módulo

1 agente > múltiplos Contexto unificado, decisões coerentes. Skills são as ferramentas, não agentes separados.
Skill = processo documentado Não é prompt avulso. É um SKILL.md com instruções, inputs e critérios de qualidade.
Organização em categorias content/, analytics/, operations/, research/. Uma skill por processo, não uma skill "youtube" genérica.
Todo repetitivo vira skill Se você faz mais de 2 vezes — documenta, testa, vira skill. Consistência > improviso.
Backup no Git Skills são ativos. Repositório privado, commit regular ou cron automático.
Terceiros: leia o código ClawHub verificado é confiável, mas sempre revise scripts antes de instalar. Acesso ao workspace é total.
