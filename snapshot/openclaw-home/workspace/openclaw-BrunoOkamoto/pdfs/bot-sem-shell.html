01 Sintomas

Como saber que você está com esse problema

⚠️ Sintomas Clássicos
  • O agente diz: "Não consigo executar comandos no seu sistema"
  • O agente responde texto fluentemente, mas qualquer tarefa de sistema falha
  • Erros como exec: not in allowlist, Permission denied, Sandbox restriction
  • Bot adicionado ao grupo — não responde nada (nem erros)
🧪
Teste rápido

Peça pro agente executar echo "SHELL_TEST_OK". Se receber o texto de volta, shell OK. Se der erro — você tem um dos problemas abaixo.

02 As 5 Causas
⚡ CAUSA 0 — MAIS COMUM · NOVO 3.2

tools.profile = "messaging" NOVO

Desde a 3.2, a causa #1 de "bot não faz nada" é o tools.profile configurado como messaging. Esse perfil otimiza o agente para responder texto no Telegram/WhatsApp — mas bloqueia exec e acesso ao sistema.

{ "agents": { "main": { "tools": { "profile": "messaging" // ← PROBLEMA }}} }
// CORREÇÃO: trocar para "default" ou remover a chave { "agents": { "main": { "tools": { "profile": "default" // ← CORREÇÃO }}} }
CAUSA 1

exec não está no allowlist

O agente tem uma lista de ferramentas permitidas. Se exec não estiver nela, nenhum comando roda.

{ "agents": { "main": { "tools": { "allowlist": ["exec", "read", "write", "edit", "web_search"] }}} }
CAUSA 2

Sandbox all sem permissão correta

Com sandbox: "all", até o agente principal fica isolado. Use off pro principal ou non-main para sandbox só em sub-agentes.

ValorComportamento
offSem sandbox — acesso total ao sistema
non-mainSandbox só em sub-agentes
allSandbox em todos — inclui agente principal ⚠️

CAUSA 3

Security mode deny

Com security: "deny", o agente bloqueia qualquer execução de comandos. Mude para allowlist.

{ "agents": { "main": { "security": "allowlist" // era "deny" }} }
CAUSA 4 · GRUPO TELEGRAM

BotFather Group Privacy Mode

Bot funciona em DM mas ignora mensagens no grupo? É o Group Privacy Mode do BotFather — ativo por padrão. Com privacy ON, o bot só recebe mensagens que começam com /comando.

📋 Como desativar
  1. Abra @BotFather no Telegram
  2. Envie: /mybots → selecione seu bot
  3. Clique em "Bot Settings"
  4. Clique em "Group Privacy"
  5. Clique em "Turn off"
  6. Confirmação: "Privacy mode is disabled"
  7. ⚠️ Remova e re-adicione o bot ao grupo!
CAUSA 5 · VPS

Path do shell não encontrado

Mais comum em VPS, containers Docker e distribuições minimalistas. O OpenClaw não acha o shell.

{ "agents": { "main": { "shell": "/bin/bash" // ou "/bin/sh" em Alpine }} }

Confirme o path: rode which bash ou which sh no terminal do servidor.

03 Checklist Pós-Correção
✅ Verificação
Reiniciar gateway: openclaw gateway restart
Verificar status: openclaw gateway status
Teste básico: pedir ao agente echo "SHELL_TEST_OK"
Teste de arquivo: touch /tmp/teste-openclaw && echo "OK"
Verificar logs: openclaw gateway logs --tail 20
Testar tarefa real: "liste os arquivos do diretório atual"
Se bot de grupo: testar mensagem no grupo após re-adicionar
