As 3 Causas

Qualquer uma delas já basta para o bot ficar mudo. Podem ser as 3 ao mesmo tempo.

🔇 Causa 1 — Privacy Mode ATIVADO no BotFather
Por padrão, o Telegram cria bots com "privacy mode" ATIVADO. O bot NÃO RECEBE mensagens do grupo — só recebe mensagens que começam com /comando ou que mencionam diretamente o bot com @username. Mais comum.
⚙️ Causa 2 — groupPolicy não configurado no OpenClaw
Mesmo que o bot receba as mensagens, o OpenClaw tem uma camada de segurança própria: por padrão, ele não processa mensagens de grupos a menos que você configure explicitamente.
🔐 Causa 3 — Chat ID não está no allowlist
Se você usou groupPolicy = allowlist, precisa adicionar o chat ID do grupo específico. Sem o ID correto, o OpenClaw ignora as mensagens mesmo com tudo configurado.

🔑 Teste Rápido: Qual é a sua causa?

🔍
Envie /start no grupo:

• Bot responde ao /start mas ignora mensagens normais → Causa 1 (Privacy Mode)
• Bot não responde nada nem ao /start → pode ser Causa 2 ou 3
• Bot funciona em DM, mudo total no grupo → Seguir fluxograma

02
Fix 1 — Privacy Mode
1
BotFather — Desativar Privacy Mode
CAUSA 1
  1. Abra o Telegram e fale com @BotFather
  2. Envie: /setprivacy
  3. BotFather vai listar seus bots — selecione o seu
  4. Escolha: Disable
  5. Confirmação: "Success! Privacy mode is disabled."
🚨
CRÍTICO — Remova e re-adicione o bot ao grupo!

Após mudar o privacy mode, você precisa remover o bot do grupo e adicionar de volta. O Telegram só aplica a nova configuração quando o bot entra no grupo. Se o bot já estava, a mudança não vai funcionar.

03
Fix 2 — groupPolicy
2
Configurar groupPolicy no OpenClaw
CAUSA 2 + 3
# Política recomendada: allowlist (só grupos autorizados) openclaw config set channels.telegram.groupPolicy allowlist # Adicionar chat ID do grupo (veja seção abaixo para descobrir o ID) openclaw config set channels.telegram.groupAllowlist "[-100XXXXXXXXXX]" # Se tiver mais de um grupo: openclaw config set channels.telegram.groupAllowlist "[-100111111111, -100222222222]" # Para permitir qualquer grupo (menos seguro, não recomendado): openclaw config set channels.telegram.groupPolicy open
💡
Como descobrir o Chat ID do grupo

Adicione o bot ao grupo, envie qualquer mensagem e verifique os logs:

openclaw gateway logs | grep "chat_id" # Saída: Received message from chat_id: -1001234567890, user: João # Chat IDs de grupos começam com -100
🔎
Alternativa: bot @userinfobot

Adicione @userinfobot ao grupo, mande /start, ele retorna o ID do grupo automaticamente.

dmPolicy vs groupPolicy

Confusão comum — dois guardiões independentes:

dmPolicy groupPolicy
Controla Mensagens diretas (DM/privadas) Mensagens em grupos
Default allowlist não configurado
Escopo 1-para-1 com você Muitos usuários no chat
Risco Alguém comandar seu agente Bot ativo em grupos não autorizados

07
Erros Comuns
Bot responde /start mas ignora mensagens normais
Causa: Privacy mode ATIVADO. Fix: BotFather → /setprivacy → Disable → remover e re-adicionar bot ao grupo.
Bot não responde nada no grupo
Causa: groupPolicy não configurado. Fix: openclaw config set channels.telegram.groupPolicy allowlist
groupPolicy configurado mas ainda não responde
Causa: Chat ID não está no allowlist. Fix: Adicione o ID correto. Verifique via logs: openclaw gateway logs | grep chat_id
Após mudança de privacy mode, ainda mudo
Bot ainda está no grupo com a config antiga. Fix: Remover o bot do grupo e adicionar de volta — obrigatório.
Chat ID inválido
Copiou ID errado. Verifique nos logs: openclaw gateway logs | grep chat_id. IDs de grupo começam com -100.
