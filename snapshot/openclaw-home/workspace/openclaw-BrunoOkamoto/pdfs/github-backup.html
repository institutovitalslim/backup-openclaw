2🔑 Como criar o Personal Access Token (PAT)

O token aparece uma única vez. Copie e guarde em local seguro (ex: 1Password) antes de fechar a página.
  1. Acesse: github.com/settings/tokens/new (ou GitHub → Settings → Developer settings → Personal access tokens → Tokens classic)
  2. Note: dê um nome descritivo, ex: openclaw-backup-vps
  3. Expiration: recomendo No expiration (ou 1 ano se preferir renovar)
  4. Scopes — marque apenas:
    • ✅ repo — acesso completo a repos (inclui privados)
    • ✅ workflow — se quiser automatizar GitHub Actions
  5. Clique Generate token → copie o token gerado (começa com ghp_...)
  6. Salve no 1Password ou passe pro agente guardar via prompt

Guardar o token com o agente

💬 Prompt
Guarda esse token do GitHub no 1Password com o nome "GitHub PAT - OpenClaw Backup": ghp_xxxxxxxxxxxx
🔐 Nunca faça isso
  • Não commite o token em nenhum arquivo do repo
  • Não cole o token em mensagens de grupo
  • Não deixe em .env ou arquivos de texto solto

3⚙️ Setup manual via terminal (admins)

Essa seção é para quem prefere fazer no terminal. Usuários finais podem pular.

3.1 Inicializar o repositório

# Entrar no workspace
cd ~/.openclaw/workspace-curso-openclaw

# Inicializar git (se ainda não tiver)
git init
git branch -M main

# Conectar ao repositório remoto
git remote add origin https://github.com/SEU_USER/openclaw-backup.git

# Configurar usuário
git config user.name "Seu Nome"
git config user.email "seu@email.com"

3.2 Criar .gitignore inteligente

cat > .gitignore <<'EOF'
# Nunca commitar segredos ou dados temporários
*.env
*.key
*.pem
node_modules/
.DS_Store
__pycache__/
*.log
tmp/
EOF

3.3 Configurar autenticação com o token

# Opção 1: via URL (mais simples)
git remote set-url origin https://ghp_SEU_TOKEN@github.com/SEU_USER/openclaw-backup.git

# Opção 2: via Git credential store
git config credential.helper store
# Na primeira operação de push, informe: user=SEU_USER, senha=SEU_TOKEN

# Opção 3: via variável de ambiente (mais seguro em scripts)
export GITHUB_TOKEN=$(op item get "GitHub PAT" --field credential --reveal)

3.4 Primeiro backup

git add -A
git commit -m "backup: inicial - $(date '+%Y-%m-%d %H:%M')"
git push -u origin main

3.5 Script de backup (salvar como backup.sh)

#!/bin/bash
set -e
WORKSPACE="$HOME/.openclaw/workspace-curso-openclaw"
DATE=$(date '+%Y-%m-%d %H:%M')

cd "$WORKSPACE"
git add -A
git diff --cached --quiet && echo "Nada novo para commitar" && exit 0
git commit -m "backup: $DATE"
git push origin main
echo "✅ Backup concluído: $DATE"
chmod +x backup.sh

3.6 Criar cron via terminal

# Editar crontab
crontab -e

# Adicionar linha (todo dia à meia-noite)
0 0 * * * /root/.openclaw/workspace-curso-openclaw/backup.sh >> /tmp/backup.log 2>&1

# Todo dia às 23h (horário UTC = 20h Brasília)
0 20 * * * /root/.openclaw/workspace-curso-openclaw/backup.sh

4⏰ Cron de backup via OpenClaw (recomendado)

O OpenClaw tem sistema de cron nativo. É a forma mais integrada — o agente commita, faz push e te avisa no Telegram.

Como funciona

Você pede pro agente criar o cron via chat. Ele agenda um job isolado que roda diariamente, faz git add + commit + push, e manda uma mensagem confirmando o backup.

💬 Prompt completo para configurar
Cria um cron job pra fazer backup do workspace no GitHub todo dia às 23h no horário de Brasília. O job deve: fazer git add de tudo, commitar com a data de hoje, fazer push, e me mandar uma mensagem confirmando. Usa sessionTarget isolated e manda o resultado aqui no Telegram.
💬 Prompt — verificar status
Lista todos os cron jobs ativos e me mostra quando foi o último backup no GitHub
💬 Prompt — ver histórico de execuções
Mostra o histórico das últimas 5 execuções do cron de backup
💬 Prompt — rodar agora
Roda o cron de backup agora sem esperar o horário agendado
💬 Prompt — pausar
Pausa o cron de backup automático temporariamente
💬 Prompt — deletar
Deleta o cron de backup do GitHub permanentemente

6🚀 Migrar para novo servidor com o backup

Se precisar mudar de VPS ou reinstalar tudo, o backup no GitHub garante que o agente volta exatamente como estava.

💬 Prompt para migrar
Acabei de instalar o OpenClaw num servidor novo. Me ajuda a restaurar o workspace do backup em github.com/meuuser/openclaw-backup

Passos manuais

# No novo servidor, após instalar OpenClaw
git clone https://ghp_TOKEN@github.com/SEU_USER/openclaw-backup.git \
  ~/.openclaw/workspace-curso-openclaw

# Verificar que tudo chegou
ls ~/.openclaw/workspace-curso-openclaw/
cat ~/.openclaw/workspace-curso-openclaw/MEMORY.md
💡 O agente vai acordar com toda a memória intacta — MEMORY.md, arquivos de contexto, configurações.
