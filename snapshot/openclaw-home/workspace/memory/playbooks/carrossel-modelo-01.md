# carrossel-modelo-01.md

## Objetivo

Definir o padrao visual, editorial e estrutural do `Modelo 01` para carrosseis da Clara.

A regra principal e: quando Tiaro pedir um carrossel no `Modelo 01`, Clara deve seguir esse formato com alta fidelidade e nao improvisar um layout generico.

## Estrutura Obrigatoria

1. Capa
- Foto real, forte e aspiracional.
- Headline grande com promessa clara.
- Subtitulo curto com credencial cientifica ou contexto de autoridade.

2. Pilar 1
- Um conceito por slide.
- Titulo curto e forte.
- Dado principal com numero especifico.
- Explicacao curta e clara.
- Referencia cientifica no rodape.

3. Pilar 2
- Mesmo formato do slide anterior.

4. Pilar 3
- Mesmo formato do slide anterior.

5. Minha pratica
- Quebra visual em fundo claro.
- Traducao do conteudo para rotina pratica.
- Bullets objetivos e aplicaveis.

6. Fechamento / CTA
- Foto humana e memoravel.
- Frase final forte.
- CTA simples, normalmente `salve este post`.

## Regras Visuais

- Formato vertical 4:5.
- Fotos reais, com cara editorial/premium.
- Nao usar visual generico de banco de imagens sem alma.
- Titulos grandes, de alto contraste.
- Texto em blocos curtos.
- Referencias discretas no rodape dos slides de prova.
- Um conceito central por slide.
- Quando houver texto sobre foto, usar contraste alto e overlay escuro quando necessario.

## Regras De Copy

- Headline curta e forte.
- Subtitulo enxuto.
- Numeros especificos quando houver dado cientifico.
- Tom de autoridade sem excesso academico.
- Frases memoraveis no fechamento.
- Evitar CTA fraco ou generico.

## Logica Editorial

- Abrir com promessa ampla.
- Provar com 3 evidencias.
- Traduzir para pratica pessoal.
- Fechar com CTA simples e memoravel.

## Quando Usar Gemini

- Se Clara precisar gerar variacoes de texto, angulos, ganchos, headlines ou estrutura base para carrossel, Gemini esta liberado na VPS.
- Se o pedido for de criacao visual final, Clara deve respeitar primeiro o modelo visual do usuario e so usar Gemini como apoio de texto, ideacao ou refinamento, nao como desculpa para fugir do padrao.

## Regra Operacional

- Se Tiaro mencionar `Modelo 01`, assumir automaticamente este playbook.
- Se houver imagens de referencia enviadas por Tiaro, elas viram a fonte primaria do estilo.
- Se houver conflito entre o playbook e a referencia enviada pelo Tiaro, a referencia enviada pelo Tiaro vence.
