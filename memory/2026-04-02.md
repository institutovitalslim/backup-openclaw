- Tiaro autorizou autonomia para evoluir continuamente as rotinas e análises no Quarkclinic, cruzando dados de pacientes/agendamentos/prontuário com o contexto consolidado de WhatsApp para manter memória viva e histórico por paciente.
- No Quarkclinic, para reagendamentos futuros, Tiaro pediu para usar transferência direta do atendimento/agendamento em vez de cancelar e criar novo registro quando isso for possível na API.
- Na investigação da bridge Z-API -> Clara, ficou validado que o caminho confiável de invocação é o OpenClaw Gateway HTTP (`/v1/responses` ou `/v1/chat/completions`) com `x-openclaw-session-key` estável por telefone e `x-openclaw-model: openai/gpt-5.4`. O falso sintoma de "prompt contaminado" vinha do fallback automático de modelo: sem forçar um modelo disponível, o runtime injetava `Continue where you left off...`. Foi criado o esqueleto operacional em `ops/zapi_bridge/zapi_clara_bridge.py` e documentado em `ops/zapi_bridge/README.md`, preservando fan-out opcional para o Apps Script e resposta de volta via Z-API.
