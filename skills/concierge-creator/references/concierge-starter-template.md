# Concierge Starter Template

Use this starter when creating a new concierge from zero.

## Files to create
- `concierge_system_prompt.md`
- `concierge_brain.md`
- `concierge_learning_log.md`
- `WhatsApp.md` or channel-specific memory file
- `concierge_qa_scenarios.md`
- `concierge_qa_report.md`
- `concierge_preproduction_plan.md`

## 1. Prompt-base template

### Identity
Who the concierge is.

### Mission
What action it must maximize.

### Tone
How it should sound.

### Core flow
How it receives, qualifies, explains, handles objections, and advances.

### Price rules
How to answer price and protect value.

### Boundaries
What it must never promise, say, or improvise.

### Escalation / handoff
When a human must step in.

## 2. Brain template
Document:
- recurring pains
- recurring objections
- what converts
- what hurts trust
- price sensitivities
- brand truths
- evolution rules

## 3. Learning log template
For each completed conversation record:
- outcome
- main pain
- main objection
- what worked
- what failed
- phrase worth keeping
- prompt update suggested
- brain update suggested

## 4. Channel-memory template
Document channel-specific behavior:
- common openings
- message rhythm
- what sounds natural
- what sounds robotic
- hot-lead signals
- danger signals

## 5. QA battery template
Include scenarios for:
- direct price
- objection to price
- hot lead
- cold lead
- logistics
- prior frustration
- emotional lead
- booking-ready lead

## 6. Pre-production plan template
Document:
- controls
- monitoring rules
- pause criteria
- handoff rules
- daily review loop
- weekly refinement loop

## 7. Launch rule
Do not call a concierge “ready” until it has:
- prompt-base
- brain
- learning log
- QA battery
- QA report
- control logic if automated
- pre-production plan
